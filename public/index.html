Automated Pipeline Walkthrough

When a developer pushes changes to the repository, our automated GitHub Actions workflow triggers the following rigorous pipeline stages:

1

Rigorous Testing

Jest and Supertest inspect key route status codes, health responses, and SHA version payloads to protect against regressions.

2

Code Quality Guard

ESLint parses files to identify syntax discrepancies, trailing styles, and runtime flaws before proceeding.

3

Bake Commit SHA

The workflow reads the triggering commit SHA, passing it as a build argument (`GIT_SHA`) directly into the Dockerfile compiler.

4

Multi-Stage Docker

The app compiles within a minimal Alpine environment, separating builders from runtime nodes to produce an ultra-slim container image.

5

Registry Push

Authenticates and pushes two fully tagged builds (`latest` and `$SHA`) directly to your public Docker Hub registry.

6

Render Deploy Hook

An automated POST request triggers Render to retrieve the fresh Docker image, initiating zero-downtime rolling service deployment.
